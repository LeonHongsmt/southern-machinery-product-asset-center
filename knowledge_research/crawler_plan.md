# 批量抓取产品资料爬虫方案

## 目标与边界

本方案用于规划如何批量发现、抓取、识别和整理产品相关资料文件，包括 PDF、图片、说明文档及其他可下载资源。本文只描述执行方案，不包含业务代码、依赖安装或具体框架实现。

核心目标：

- 尽可能完整地发现产品资料链接。
- 将文件按产品型号、文件类型、来源页面和用途进行结构化归档。
- 对无法识别型号的文件保留原始信息，进入人工复核队列。
- 输出 JSON、CSV、Excel 三类结果，便于后续建库、前端展示和人工校验。

## 1. 如何抓取全部 PDF 链接

### 入口来源

PDF 链接通常不会只出现在产品详情页，建议从多个入口统一发现：

- 官网首页、产品列表页、分类页、搜索结果页。
- 产品详情页中的下载区、规格书区、资料区、支持区。
- 独立的下载中心、资源中心、文档中心、技术支持页面。
- sitemap.xml、robots.txt 中暴露的站点地图。
- 页面中的分页、筛选结果、异步加载接口返回内容。
- 页面源代码中的 JSON-LD、内嵌 JSON、配置对象。

### 链接发现策略

优先从 HTML 中提取所有 `<a>` 标签的 `href`，再结合页面文本判断是否为 PDF 资料：

- URL 后缀为 `.pdf`。
- URL 查询参数中包含 PDF 文件名，例如 `file=xxx.pdf`、`download=xxx.pdf`。
- 响应头 `Content-Type` 为 `application/pdf`。
- 链接文本包含 `PDF`、`Datasheet`、`Spec`、`Manual`、`Catalog`、`Brochure`、`说明书`、`规格书`、`资料下载` 等关键词。
- 下载按钮跳转到中转地址时，需要跟随重定向并记录最终 URL。

### 去重规则

PDF 链接建议按以下层级去重：

- 规范化后的最终 URL。
- 去除跟踪参数后的 URL，例如 `utm_*`、`from`、`spm` 等。
- 文件名加文件大小。
- 文件内容哈希值，适用于下载后复核。

## 2. 如何抓取全部图片链接

### 图片来源

图片链接可能来自多个位置：

- `<img>` 标签的 `src`、`srcset`、`data-src`、`data-original`、`data-lazy`。
- `<picture>`、`source` 标签中的响应式图片。
- CSS 背景图中的 `url(...)`。
- 页面内嵌 JSON 中的产品主图、相册图、缩略图字段。
- Open Graph、Twitter Card 等元信息中的预览图。
- 产品轮播图、详情图、规格图、安装示意图。
- 异步接口返回的图片数组。

### 图片筛选规则

为了避免抓到大量无关图标和装饰图，需要设置过滤规则：

- 保留常见图片格式：`.jpg`、`.jpeg`、`.png`、`.webp`、`.gif`、`.avif`、`.svg`。
- 优先保留路径中包含 `product`、`products`、`gallery`、`upload`、`images`、`media`、`detail`、`thumbnail` 等关键词的图片。
- 排除明显无关资源，例如 `logo`、`icon`、`sprite`、`favicon`、`banner-ad`、`tracking`。
- 对尺寸过小的图片降权或排除，例如宽高小于 200px 的装饰图。
- 对同一张图片的不同尺寸版本，保留最大图，同时记录所有候选 URL。

### 图片关联产品

图片链接应尽量绑定到发现它的产品页面：

- 若图片来自产品详情页，默认关联当前页面识别出的产品型号。
- 若图片 URL 或文件名中包含型号，优先使用文件自身识别结果。
- 若图片来自列表页，需要结合图片附近的产品卡片标题、链接和 alt 文本识别。
- 若图片无法识别型号，保留来源页面、alt 文本、附近文本和原始 URL。

## 3. 如何抓取全部说明文档链接

说明文档不一定是 PDF，也可能是网页、压缩包、Office 文件或在线帮助页面。建议将“说明文档”定义为所有面向用户或技术人员解释产品使用、安装、维护、选型和规格的资料。

### 文档类型范围

应纳入抓取范围的说明文档包括：

- 产品说明书、用户手册、安装手册、维护手册。
- 数据手册、规格书、选型指南、白皮书。
- 产品目录、宣传册、认证文件。
- 在线帮助页面、FAQ、知识库文章。
- Word、Excel、PowerPoint 文档。
- ZIP、RAR、7Z 等压缩包中的资料集合。

### 文档链接识别

通过以下信号判断说明文档链接：

- 文件后缀为 `.pdf`、`.doc`、`.docx`、`.xls`、`.xlsx`、`.ppt`、`.pptx`、`.zip`、`.rar`、`.7z`、`.txt`、`.html`、`.htm`。
- 链接文本包含 `Manual`、`Guide`、`Instructions`、`Datasheet`、`Specification`、`Catalog`、`Download`、`说明书`、`手册`、`安装`、`维护`、`规格`、`认证`、`下载`。
- 页面结构位于 `Downloads`、`Support`、`Resources`、`Documents`、`资料下载`、`技术支持`、`文档中心` 等模块下。
- 在线文档页面存在目录结构、步骤说明、参数表、FAQ 或版本号。

### 在线文档处理

对非文件型在线说明文档，应保留：

- 文档页面 URL。
- 页面标题。
- 关联产品型号。
- 文档类型判断结果。
- 页面摘要或主要文本所在区域。
- 来源入口页面。
- 抓取时间。

## 4. 如何识别文件类型

文件类型识别建议采用多重判断，避免只依赖 URL 后缀。

### 判断优先级

1. HTTP 响应头 `Content-Type`。
2. 文件真实内容的魔数或文件签名。
3. URL 路径和文件扩展名。
4. `Content-Disposition` 中的文件名。
5. 链接文本、按钮文本和上下文文案。
6. 来源页面模块标题。

### 常见类型映射

| 文件类型 | 常见后缀 | 常见 Content-Type | 用途 |
| --- | --- | --- | --- |
| PDF | `.pdf` | `application/pdf` | 规格书、说明书、认证文件 |
| 图片 | `.jpg`、`.png`、`.webp`、`.svg` | `image/*` | 产品图、详情图、示意图 |
| Word | `.doc`、`.docx` | `application/msword`、Office MIME | 说明文档、模板 |
| Excel | `.xls`、`.xlsx` | Excel MIME | 参数表、清单 |
| PowerPoint | `.ppt`、`.pptx` | PowerPoint MIME | 产品介绍 |
| 压缩包 | `.zip`、`.rar`、`.7z` | `application/zip` 等 | 批量资料包 |
| 网页文档 | `.html`、无后缀 | `text/html` | 在线说明、FAQ |
| 文本 | `.txt`、`.csv` | `text/plain`、`text/csv` | 清单、参数 |

### 异常处理

- URL 无后缀但响应头明确时，以响应头为准。
- 响应头为通用二进制流时，继续检查文件签名和文件名。
- 扩展名与内容类型冲突时，标记为 `type_conflict`，进入复核。
- 下载失败或无法访问时，保留 URL 并标记为 `unverified`。

## 5. 如何从文件名或路径中识别产品型号

产品型号识别应结合文件名、URL 路径、页面标题和页面上下文，而不是只靠单一正则。

### 识别来源优先级

1. 产品详情页的结构化字段，例如标题、型号字段、面包屑、页面 schema。
2. 文件名中的型号。
3. URL 路径中的型号。
4. 链接文本中的型号。
5. 图片 alt、title 或附近文本。
6. 来源页面标题或产品卡片标题。
7. 站内搜索或产品列表中建立的型号索引。

### 型号候选特征

常见产品型号通常具备以下特征：

- 包含字母和数字组合，例如 `ABC123`、`X-200`、`M12-8P`。
- 包含连字符、下划线、斜杠或点号，例如 `AB-1000`、`X_20`、`V2.1`。
- 位于文件名开头或靠近关键词，例如 `AB-1000_datasheet.pdf`。
- 位于路径层级中，例如 `/products/ab-1000/downloads/spec.pdf`。
- 与产品标题、页面 H1 或列表卡片名称一致。

### 清洗规则

识别型号前应对文本做标准化：

- URL 解码。
- 将 `%20`、空格、下划线统一处理为可比较形式。
- 去除文件扩展名。
- 去除语言、版本、日期、文档类型词，例如 `en`、`cn`、`v1`、`2025`、`manual`、`datasheet`。
- 保留大小写原文，同时生成统一大写或统一小写的匹配键。

### 匹配策略

建议先建立产品型号词典，再从文件信息中做匹配：

- 从产品列表页和详情页提取所有产品名称与型号，形成 `product_index`。
- 对文件名、路径、链接文本、来源页面标题进行精确匹配。
- 对大小写、分隔符差异做宽松匹配。
- 对近似命中结果打分，而不是直接覆盖。
- 当多个型号同时命中时，保留全部候选，并选择最高分作为 `primary_model`。

## 6. 无法识别型号的文件如何保留

无法识别型号的文件不能丢弃，应完整保留并进入待复核流程。

### 保留字段

对无法识别型号的文件，至少保留：

- 原始 URL。
- 最终 URL。
- 文件名。
- 文件类型。
- 文件大小。
- 来源页面 URL。
- 来源页面标题。
- 链接文本。
- 附近文本。
- 所在模块标题。
- 抓取时间。
- HTTP 状态码。
- 失败或无法识别原因。
- 候选型号列表。

### 分类方式

可将无法识别型号的文件分为：

- `unknown_model`：文件可访问，但无法判断产品型号。
- `ambiguous_model`：命中多个产品型号，无法确定主型号。
- `generic_asset`：通用资料，例如总目录、品牌手册、通用安装指南。
- `site_asset`：站点装饰资源或公共图片。
- `failed_fetch`：链接发现成功但文件验证失败。

### 后续处理

- 输出单独的待复核清单。
- 支持人工补充产品型号。
- 人工确认后回写映射表，供后续增量抓取复用。
- 对通用资料允许关联到产品系列、分类或品牌，而不是强行绑定单个型号。

## 7. 如何输出 JSON、CSV、Excel

### JSON 输出

JSON 适合后续程序消费和保留层级结构。建议输出两个层级：

- 产品维度：每个产品包含图片、PDF、说明文档、其他资源数组。
- 文件维度：每个文件是一条独立记录，包含来源、类型、型号识别结果和抓取状态。

建议字段：

| 字段 | 说明 |
| --- | --- |
| `id` | 文件记录唯一 ID |
| `source_page_url` | 来源页面 |
| `source_page_title` | 来源页面标题 |
| `raw_url` | 原始链接 |
| `final_url` | 跟随重定向后的最终链接 |
| `file_name` | 文件名 |
| `file_type` | 文件类型 |
| `mime_type` | 响应 MIME 类型 |
| `file_size` | 文件大小 |
| `primary_model` | 主产品型号 |
| `model_candidates` | 候选型号列表 |
| `confidence` | 型号识别置信度 |
| `document_category` | 文档类别 |
| `language` | 语言 |
| `version` | 版本 |
| `status` | 抓取状态 |
| `error_reason` | 错误原因 |
| `fetched_at` | 抓取时间 |

### CSV 输出

CSV 适合人工筛选、批量导入和差异比较。由于 CSV 不适合嵌套结构，建议将数组字段扁平化：

- `model_candidates` 使用分号分隔。
- 多个来源页面可输出多行，或使用分号分隔。
- 图片多尺寸候选可以输出主图 URL，完整候选保留在 JSON 中。

建议至少输出以下 CSV：

- `files_all.csv`：全部文件清单。
- `files_unknown_model.csv`：无法识别型号清单。
- `products_assets.csv`：产品与资源关联表。
- `crawl_errors.csv`：抓取失败和验证失败清单。

### Excel 输出

Excel 适合人工审核，建议一个工作簿包含多个工作表：

- `全部文件`：完整文件记录。
- `产品资源汇总`：按产品型号聚合的资源数量。
- `无法识别型号`：待人工复核。
- `失败链接`：失败原因、状态码、重试次数。
- `重复文件`：重复 URL 或重复内容。
- `字段说明`：解释每个字段含义。

Excel 中建议增加筛选、冻结首行、按状态或置信度排序，方便人工处理。

## 8. 推荐的爬虫执行流程

### 阶段一：站点发现

- 读取种子 URL。
- 解析 sitemap.xml 和 robots.txt。
- 抓取首页、产品分类页、下载中心、支持中心。
- 建立待抓取页面队列。

### 阶段二：产品索引建立

- 抓取产品列表页和产品详情页。
- 提取产品名称、型号、分类、详情页 URL。
- 建立产品型号词典和别名词典。
- 保存产品页面与型号的映射关系。

### 阶段三：资源链接发现

- 从页面 HTML、元信息、内嵌 JSON、异步接口响应中提取文件和图片链接。
- 规范化相对路径、协议相对路径和重定向链接。
- 记录每个链接的来源页面、链接文本、上下文和发现方式。

### 阶段四：文件验证与类型识别

- 对候选链接发起轻量验证请求。
- 读取响应头、最终 URL、文件大小、MIME 类型。
- 必要时下载文件头部或完整文件用于进一步识别。
- 标记文件类型、文档类别和可访问状态。

### 阶段五：产品型号匹配

- 使用产品型号词典匹配文件名、路径、链接文本和来源页面。
- 为每个候选型号计算置信度。
- 输出主型号、候选型号和匹配证据。
- 对低置信度或多候选结果进入人工复核。

### 阶段六：去重与归档

- 按规范化 URL、文件名、文件大小和内容哈希去重。
- 保留重复来源关系，不丢失来源页面。
- 将文件归入产品、产品系列、通用资料或未知型号分类。

### 阶段七：结果输出

- 输出 JSON 原始结构化数据。
- 输出 CSV 扁平清单。
- 输出 Excel 人工审核表。
- 输出日志、错误清单和统计摘要。

### 阶段八：人工复核与增量更新

- 人工处理无法识别型号和冲突型号。
- 建立人工确认映射表。
- 后续增量抓取时优先复用映射表。
- 对新增、变更、失效链接生成差异报告。

## 9. 失败重试和日志方案

### 失败类型

需要区分不同失败原因，避免无意义重试：

- 网络超时。
- DNS 或连接失败。
- HTTP 429 限流。
- HTTP 403 权限限制。
- HTTP 404 资源不存在。
- HTTP 5xx 服务端错误。
- 重定向循环。
- 文件类型不匹配。
- 文件损坏或下载不完整。
- 页面结构变化导致解析失败。

### 重试策略

建议按失败类型设置不同策略：

| 失败类型 | 是否重试 | 建议策略 |
| --- | --- | --- |
| 超时、连接中断 | 是 | 指数退避，最多 3 次 |
| HTTP 429 | 是 | 延长等待时间，降低并发 |
| HTTP 5xx | 是 | 延迟重试，最多 3 次 |
| HTTP 403 | 视情况 | 检查访问策略，不反复请求 |
| HTTP 404 | 否 | 标记失效链接 |
| 类型不匹配 | 否 | 标记 `type_conflict` |
| 重定向循环 | 否 | 标记 `redirect_loop` |
| 文件损坏 | 是 | 重下 1 次，仍失败则复核 |

### 日志分层

日志建议拆成三类：

- 运行日志：记录任务开始、结束、队列数量、并发数、耗时。
- 链接日志：记录每个 URL 的发现、验证、下载、状态码和重试次数。
- 解析日志：记录产品型号识别、文件类型识别、置信度和异常原因。

### 关键日志字段

每条日志建议包含：

- `run_id`：本次任务 ID。
- `timestamp`：日志时间。
- `level`：日志级别。
- `event`：事件名称。
- `url`：当前处理 URL。
- `source_page_url`：来源页面。
- `status_code`：HTTP 状态码。
- `retry_count`：重试次数。
- `duration_ms`：请求耗时。
- `file_type`：识别出的文件类型。
- `primary_model`：识别出的主型号。
- `confidence`：置信度。
- `error_reason`：错误原因。

### 统计摘要

每次任务完成后输出摘要：

- 抓取页面数量。
- 发现文件链接数量。
- 成功验证文件数量。
- PDF、图片、说明文档数量。
- 已识别型号文件数量。
- 无法识别型号文件数量。
- 失败链接数量。
- 重复文件数量。
- 新增、更新、失效文件数量。

## 合规与访问控制建议

- 遵守目标网站 robots.txt 和服务条款。
- 控制请求频率，避免对目标站点造成压力。
- 记录来源 URL，保证资料可追溯。
- 不绕过登录、验证码、付费墙或访问限制。
- 对版权资料只做内部资料索引和合规使用。

## 推荐最终产物

建议一次完整抓取后产出以下文件：

- `products.json`：产品维度结构化数据。
- `files_all.json`：文件维度完整数据。
- `files_all.csv`：全部文件扁平清单。
- `files_unknown_model.csv`：无法识别型号清单。
- `crawl_errors.csv`：失败链接清单。
- `product_assets.xlsx`：人工审核工作簿。
- `crawl_summary.md`：本次抓取统计摘要。

## 验收标准

- 能覆盖 PDF、图片、说明文档和其他可下载资料。
- 每个文件都能追溯到来源页面。
- 可识别型号的文件具备型号、置信度和匹配依据。
- 无法识别型号的文件不会丢失，并能进入人工复核。
- 输出格式同时满足程序处理和人工审核。
- 失败、重复、冲突和低置信度数据都有明确标记。
